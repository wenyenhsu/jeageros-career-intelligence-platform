Monitoring Parameter Guide

How crawl runs, pipeline events, status counts, and failures are reported.

Monitoring Sources

Monitoring Flow

The dashboard reads persisted crawl runs and pipeline logs. It does not run crawler logic directly.

  1. A source run creates or reuses a CrawlRun.
  2. Each pipeline service writes PipelineLog events.
  3. The live status panel polls the run status endpoint.
  4. The Monitoring page summarizes the selected run or latest logs.
  5. Failures keep their reason text so partial crawls can be debugged later.

Dashboard Sections

The Monitoring page is ordered from run-level status to detailed event history.

Section Purpose Primary Source
Latest Crawl Run Shows the newest or selected run status, time window, progress, and totals. CrawlRun
Pipeline Flow Shows individual events in the order they were recorded. PipelineLog
Job Archive Archives older jobs, stores an export snapshot, and restores archived jobs when needed. JobArchiveRun and JobPost.status
Top Error Sources Ranks sources that recently produced failures. PipelineLog.source
Recent Failures Lists the newest failed events with readable reason text. PipelineLog.error_text and metadata

Latest Crawl Run Fields

Parameter Meaning How to Read It
Status Current lifecycle state for the crawl run. PENDING, RUNNING, SUCCESS, FAILED, or ABORTED.
Started When the crawl run began execution. If blank, the run was queued but did not start yet.
Finished When the crawl run reached a terminal state. Blank means the run is still pending or running.
Processed Number of sources processed out of the total selected sources. 0 / 1 means one source is planned and none has finished.
Progress Run-level completion percentage. Usually processed_sources / total_sources. Live pages may show estimated pipeline progress while a source is mid-run.
Created Jobs inserted during this run. Useful for seeing whether the crawler found new job records.
Updated Existing jobs refreshed during this run. High values are normal when the source returns jobs already in the database.
Closed Known jobs marked closed because they were missing from the newest crawl scope. Records are preserved; they are not deleted.
Errors Failures recorded at run, source, extraction, sync, or skill pipeline level. Click through to Pipeline Flow or Recent Failures to inspect the reason.

Pipeline Flow Fields

Parameter Meaning Example
Time Local display time for when the event was recorded. Jun 17, 10:59 PM. Hover or inspect the timestamp for the full ISO value.
Step The pipeline stage that wrote the event. source_detection, listing_discovery, job_extraction, source_sync.
Status Outcome for that event. STARTED, SUCCESS, FAILED, SKIPPED, or INFO.
Duration Time spent on the step or time since the previous event. 1:26 means one minute and twenty-six seconds.
Message Human-readable summary written by the service. Finished crawling LinkedIn data career.
Reason Error detail extracted from exception text or structured metadata. HTTP Error 429: Too Many Requests.

Pipeline Step Names

Step Meaning
crawl_run Run-level lifecycle event such as queued, started, finished, failed, or aborted.
celery_task Background task queue event for scheduled or manual crawl execution.
source_detection Parser selection based on the configured source type and URL.
listing_discovery Search/listing page generation or discovery.
job_extraction Raw job extraction from listing or detail pages.
source_sync Company upsert, job upsert, duplicate prevention, and closed-job detection.
ollama_extract Ollama candidate skill extraction from normalized job or resume text.
ollama_verify Ollama verification that extracted skills are supported by the source text.
skillset_mapping Mapping verified skills to the canonical SkillSet table and keyword aliases.
skill_scoring Deterministic evidence scoring for mapped skills.
skill_attach Attaching final SkillSet records to jobs or applications.
skill_pipeline Combined status for extract, verify, map, score, and attach.
crawl_progress Progress checkpoint after a source finishes processing.
analytics_query Analytics service aggregation event.

Job Archive Fields

Parameter Meaning How to Read It
Created before User-selected age threshold in months. 3 archives jobs whose created_at is older than three months.
Cutoff Exact timestamp calculated from the age threshold. Jobs created before this timestamp are eligible for archive.
Archived Number of jobs moved to ARCHIVED status. Archived jobs remain in the database and keep their SkillSet links.
Download Exports the archive snapshot as JSON. The file includes job fields, company label, timestamps, and skill links.
Restore Moves archived jobs back to ACTIVE. Restore updates each restored job's created_at to the restore time.

Filters, Links, and Live Status

Parameter Meaning Usage
crawl_run_id Optional query parameter that limits the dashboard to one crawl run. /sources/monitoring/?crawl_run_id=123
run Short alias for crawl_run_id. /sources/monitoring/?run=123
Selected run When a run id is valid, Monitoring only shows logs for that run. Use this after clicking a crawl notification or live pipeline link.
Invalid run When a run id no longer exists, the page falls back to latest monitoring data. The warning means the filter was ignored safely.
Live status endpoint JSON endpoint used by the live pipeline card. GET /sources/runs/<id>/status/
Abort endpoint Stops a running crawl by marking the run as aborted. POST /sources/runs/<id>/abort/

Common Failure Reasons

Source and network failures

HTTP 429
The source rate-limited requests. Lower request limits, increase delay, or wait for cooldown.
Name or service not known
DNS failed inside the runtime or container.
Network is unreachable
The runtime cannot reach the internet or target host.
The read operation timed out
The source accepted the connection but did not respond before timeout.

Pipeline and Ollama failures

Ollama request failed
The Ollama service URL is unreachable, the model is missing, or the service is down.
Malformed JSON
The model response could not be parsed into the expected deterministic schema.
Skill pipeline failed
One or more extract, verify, map, score, or attach steps failed for a job.
0 created / 0 updated
The run may have found no jobs, filtered all results, or hit a source error before sync.
