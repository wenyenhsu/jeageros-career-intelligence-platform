Job Source Parameter Guide

How each source setting affects crawling, filtering, and stored job data.

Create source Sources

Configuration Flow

The form is a structured editor for the JSON fields stored on each JobSource.

  1. Choose a resource type.
  2. The form fills safe defaults for that resource.
  3. Edit the crawl and filter parameters.
  4. Saving writes normalized values into crawl_config and filter_config.
  5. The crawler reads those values during the next source run.

Resource Defaults

Resource Default Base URL Best Use
LinkedIn https://www.linkedin.com/jobs/search/ LinkedIn job search pages. Filters should be configured in the form instead of hardcoded into the URL.
Handshake https://app.joinhandshake.com/stu/postings Handshake postings. Use the same normalized filter fields where possible.
Generic HTML Custom URL required Career pages or other HTML pages that do not have a dedicated parser yet.

Source Fields

Parameter Meaning Example
Name Human-readable label for the source. LinkedIn data internship
Resource Parser family used by source detection and crawling. LinkedIn
Base URL The starting URL for the crawler. For LinkedIn, keep this general and configure filters separately. https://www.linkedin.com/jobs/search/
Enabled Only enabled sources are included in scheduled or "Run all sources" crawls. Enabled
Crawl interval minutes Intended interval between scheduled crawls for this source. 1440 for once per day

Crawl Config

Parameter Meaning Guidance
Max pages Number of search result pages to scan per run. Keep low for LinkedIn to reduce rate-limit risk.
Fetch details Controls whether the crawler opens individual job detail pages. New or missing descriptions is the safest default.
Request delay seconds Wait time between crawler requests. Use higher values when a site rate-limits aggressively.
Max search requests Maximum search/listing requests per run. Lower values reduce 429 risk but crawl fewer combinations.
Max detail requests Maximum job-detail requests per run. Use with Fetch details to control request volume.
Rate limit cooldown minutes How long the source should cool down after rate limiting. 60 is a conservative starting point.
Default job type Fallback job type used only when the page does not expose one clearly. Avoid setting this unless the source is intentionally scoped.
Rolling search Rotates through search combinations across runs instead of always starting at the first combination. Recommended when there are many locations, keywords, or job types.

Filter Config

Parameter Meaning Accepted Format
Location Locations used by search URL generation and post-crawl filtering. Comma or newline separated, such as TX, CA, United States.
Job types Allowed job types. LinkedIn maps these to its job-type filters. Full-time, Internship, Part-time, Contract, Temporary.
Workplace types Allowed workplace modes. Remote, Hybrid, On-site.
Remote only Shortcut that narrows workplace filtering to remote jobs. Use only when remote jobs are the only acceptable result.
Search keywords Terms sent to the source search engine. data engineer, backend, internship.
Include keywords Post-crawl text filter. Matching jobs should contain at least one of these terms. python, django, api.
Exclude keywords Terms that should disqualify a job after extraction. senior, staff, principal.
Target companies Optional company allowlist. Leave empty to accept any company that matches the other filters.

Practical Defaults

Low-risk LinkedIn crawl

Good for avoiding rate limits while still collecting useful jobs.

Max pages: 1
Max search requests: 5
Max detail requests: 5
Request delay seconds: 5-10
Fetch details: New or missing descriptions
Rolling search: Enabled

Internship-focused filter

Good when the source should only collect internship-like roles.

Location: TX, CA, United States
Job types: Internship
Workplace types: Remote, Hybrid, On-site
Search keywords: data engineer, backend, internship
Target companies: leave empty for no company limit
